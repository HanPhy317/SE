# 校园跑腿平台 — 用户使用手册

> 前后端分离架构 | FastAPI + Vue 3 | MySQL 8.0 | JWT 认证

---

## 一、项目概览

校园跑腿平台是一个面向高校学生的互助跑腿 Web 应用，支持**普通用户发布跑腿订单**和**骑手抢单配送**两种角色。

**核心功能：**
- 用户注册/登录（用户名或手机号，JWT Token 认证）
- 4 种跑腿类型：外卖、快递、代买、自定义
- 骑手注册、抢单大厅、接单配送
- 订单状态机：`待接单 → 已接单 → 配送中 → 已送达 → 已完成`
- 自动结算：收货确认后，发布者扣款 + 骑手入账 + 双流水记录
- **骑手抢单并发安全**：asyncio 应用锁 + MySQL 行锁双重保护
- 移动端优先的响应式 UI（手机/平板/桌面自适应）

### 2.2 创建数据库

登录 MySQL，执行以下 SQL：

```sql
CREATE DATABASE IF NOT EXISTS campus_errand_db
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### 2.3 修改数据库连接（如需）

编辑 [`backend/config.py`](file:///d:/pythoncode/SE/backend/config.py) 第 9 行：

```python
DATABASE_URL = "mysql+aiomysql://用户名:密码@localhost:3306/campus_errand_db"
```

### 2.4 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2.5 安装前端依赖

```bash
cd frontend

```

## 四、功能使用指南

### 4.1 用户注册

1. 打开浏览器访问 `http://localhost:5173`，自动跳转到登录页
2. 点击底部「还没有账号？立即注册」
3. 填写信息：
   - **用户名**：2-64 个字符（唯一）
   - **手机号**：11 位数字（唯一）
   - **密码**：至少 6 位
4. 点击「注册」，成功后自动登录并跳转到「我的订单」页面

> 新用户注册后系统会赠送 **100 元**初始余额用于测试。

### 4.2 用户登录

登录页面支持两种登录身份：

- **普通用户**：登录后看到「我的订单」「发布订单」「个人信息」
- **骑手**：登录后看到「订单大厅」(骑手抢单页)「个人信息」

登录凭据可以是**用户名**或**手机号**，选择对应角色后点击「登录」。

### 4.3 注册成为骑手

**前提**：必须先注册为普通用户并登录。

**步骤**：

1. 以普通用户身份登录
2. 点击导航栏「个人信息」或底部 Tab「个人信息」
3. 在个人信息页面底部，找到「成为骑手，接单赚钱！」区域
4. 输入**服务区域**（如：北校区、主校区）
5. 点击「注册骑手」

注册成功后：
- 角色变更为骑手
- 页面显示骑手信息（信用评分 5.0、服务区域、0 单）
- 可点击「前往骑手大厅」进入抢单页面

> 一个用户只能注册一次骑手，重复注册会提示「您已经是骑手了」。

### 4.4 发布订单（普通用户）

1. 登录后点击「发布订单」（桌面端顶部导航 / 手机端底部 Tab）
2. 选择订单类型，不同类型会显示不同的业务表单：

| 订单类型 | 业务字段 |
|---------|---------|
| 外卖跑腿 | 外卖描述（如"黄焖鸡米饭"）、取餐地址（如"三食堂二楼"） |
| 快递跑腿 | 快递公司（如"顺丰"）、快递单号、取件地址（如"菜鸟驿站"） |
| 代买服务 | 店铺名称（如"校园超市"）、商品名称、预估重量（如"500g"） |
| 自定义跑腿 | 详细描述、截止时间 |

3. 填写**配送地址**（送达目的地）
4. 填写**酬劳金额**（元）
5. 点击「发布订单」

发布成功后自动跳转回「我的订单」页面，订单状态为**「待接单」**。

### 4.5 查看我的订单（普通用户）

登录后首页即为「我的订单」，订单按状态分为以下区域：

| 区域 | 状态 |
|------|------|
| 待接单 | `pending` — 等待骑手接单，红色左边框 |
| 进行中 | `accepted` / `delivering` — 骑手已接单或正在配送 |
| 已送达待确认 | `delivered` — 骑手已送达，等待您确认收货 |
| 已完成 | `completed` — 已结算完成，绿色左边框 |

**可执行操作：**
- 「取消订单」：仅待接单状态可取消
- 「确认收货并结算」：骑手送达后可点击，系统自动完成扣款和入账

### 4.6 骑手抢单大厅（骑手）

**前提**：已注册为骑手。

以骑手身份登录后进入「订单大厅」，显示三个区域：

**页面顶部 — 骑手统计卡片：**
- 评分（信用分）
- 已完成单数
- 服务区域

**可接订单列表：**
- 显示所有 `pending` 状态订单
- 订单卡片有呼吸光效（黄色脉冲边框），提示可抢
- 显示订单类型、配送地址、酬劳、发布时间、业务详情
- 点击「抢单」按钮进入抢单确认页

**我的进行中订单：**
- 显示自己已接的订单
- `accepted` 状态 → 可点击「开始配送」
- `delivering` 状态 → 可点击「确认送达」

**我的已完成订单。**

### 4.7 抢单流程（关键）

1. 在骑手大厅，点击任意待接单的「抢单」按钮
2. 进入抢单确认页，查看订单详情：
   - 顶部醒目显示酬劳金额
   - 订单编号、类型、状态、配送地址、发布时间
   - 业务字段详情
3. 确认无误后，点击 **「💰 立即抢单」**（红色大按钮，有脉冲动画）

**抢单结果：**
- **成功**：弹出绿色提示「接单成功！」，自动跳转回大厅，订单出现在「我的进行中订单」
- **失败**（已被别人抢走）：弹出提示「订单已被其他骑手接走」，页面自动刷新显示最新状态，按钮变为灰色不可点击

> 系统使用**双锁机制**确保同一订单只能被一个骑手抢到，详见第五章。

### 4.8 配送与结算

骑手接单后，按以下流程操作：

```
已接单 → [开始配送] → 配送中 → [确认送达] → 已送达
                                                   ↓
                              发布者 → [确认收货并结算] → 已完成 ✅
```

- **骑手操作**：在「我的进行中订单」点击「开始配送」「确认送达」
- **发布者操作**：在「我的订单」→「已送达待确认」区域点击「确认收货并结算」

**结算规则：**
- 酬劳从发布者余额扣除
- 酬劳加入骑手关联用户余额
- 系统生成两条交易流水记录（debit + credit）
- 骑手总接单数 +1

## 五、抢单并发安全机制

为避免多个骑手同时点击抢单导致一人接多单，系统实现**双层锁**：

| 层级 | 机制 | 作用 |
|------|------|------|
| 应用层 | `asyncio.Lock`（每订单独立锁） | 保证同一订单在 Python 进程中串行处理 |
| 数据库层 | `SELECT ... FOR UPDATE`（MySQL 行锁） | 在数据库层面锁定订单行，其他事务等待 |

**抢单流程（简化）：**

```
骑手A请求接单 → 获取 asyncio.Lock → FOR UPDATE 锁行 → 检查 status=pending？
                                                     → YES: 更新 status=accepted, rider_id=A → 提交 → 释放锁
骑手B请求接单 → 等待 asyncio.Lock ... → 获取锁 → FOR UPDATE → 检查 status=accepted？→ NO → 返回「已被抢走」
```

## 六、移动端适配说明

UI 采用**移动端优先**设计，三级响应式断点：

| 屏幕宽度 | 适配行为 |
|---------|---------|
| >768px（桌面） | 顶部导航栏显示全部链接，无底部 Tab |
| ≤768px（平板/手机） | 顶部导航栏变为汉堡菜单，显示底部 Tab 导航栏 |
| ≤480px（小屏手机） | 字号缩小，内边距缩减，表单更紧凑 |

**移动端特性：**
- 底部固定 Tab 栏：常用功能一键触达
- 汉堡菜单：点击展开/收起，滚动页面自动关闭
- `env(safe-area-inset-bottom)` 适配 iPhone 刘海屏
- `-webkit-tap-highlight-color: transparent` 消除触摸高亮
- 按钮最小 44px 触摸区域

```
frontend/ (Vue 3 + Vite, :5173)
    │

    ▼

    │
    ├── routes/auth.py      → 注册/登录/个人信息/成为骑手
    ├── routes/orders.py    → 发布/接单/配送/结算/查询

    ├── models/             → ORM 数据模型
    └── utils/              → JWT 签发/校验、统一响应格式
    │
    ▼

```

## 九、常见问题

**Q：启动后端报 `Can't connect to MySQL server`？**
A：确认 MySQL 服务已启动，`campus_errand_db` 数据库已创建，`config.py` 中用户名密码正确。

**Q：前端页面空白，控制台报 CORS 错误？**
A：确保后端和前端**同时运行**，Vite 代理配置正确。后端 CORS 已放开 `localhost:5173`。

**Q：抢单提示「你不是骑手」？**
A：需要先以普通用户登录 → 个人信息 → 注册骑手 → 再回到骑手大厅。

**Q：配送状态无法更新？**
A：状态只能按顺序流转：`accepted → delivering → delivered → completed`。不能跨状态跳转。

**Q：发布者余额不足能结算吗？**
A：不能。结算时会校验余额，不足时提示「发布者余额不足」。

**Q：可以接自己发布的订单吗？**
A：不能。系统会提示「不能接自己发布的订单」。

**Q：已接单的订单能取消吗？**
A：只有 `pending`（待接单）状态可取消。`accepted` 及之后状态需要联系对方协商。
